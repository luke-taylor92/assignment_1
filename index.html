Project Idea

Overview

In the organisation I work for, we have a need for complex passwords to be sent to our technicians. My project idea is to build an app to solve an issue that we see commonly of the technicians not knowing what letters are in these complex passwords. Such as an uppercase i (I) or a lowercase L (l) and the difference between the number zero (0) and letter o (O). The app would be able to take in the password or one-time passcode sent to the technician and give a clear read out of each letter e.g. “The first character is an Uppercase Alpha, “the second character is the number seven”.

Motivation

Every day I see technicians calling into my department with issues relating to the access codes. At least a few of them each day are down to the technician not reading or interpreting these codes correctly. This amounts to a lot of lost productivity over all the technicians and helpdesk agents over the course of a day. Having a simple tool for the technicians to be able to put the code into and spell out the code for them would save time for technicians on site and reduce the number workload onto our department. There are websites that will flip the case of an input but due to the nature of our work, copying passwords into an unknown website could pose a security risk.

Description

The app will be an android app as that is the platform my organisation uses. When the technician opens the app, they would be presented with a GUI that has a description of what the app does. Below that there will be an input box and text that asks them to paste in the password or code they have been sent by the automated system. Underneath there would be a button that says submit or something similar. Once submitted a box would appear below and print out a line for every letter of the code.

For instance, If the code was AbCd1234 the output would be:
The first Character is an Uppercase ALPHA (A)
The second Character is a Lowercase BRAVO (b)
The third Character is an Uppercase CHARLIE (C)
The fourth Character is a Lowercase DELTA (d)
The fifth Character is the Number ONE (1)
The sixth character is the Number TWO (2)
The seventh Character is the Number THREE (3)
The eighth Character is the Number FOUR (4)

Once the technician has this, they will be able to input the passcode correctly into whatever piece of equipment they are working with. The app would be built with capability for up to 24 characters as that leaves some room for longer codes to be entered if needed. The app could also be built with non-alphanumeric characters such as the ampersand (&) or backslash (\).

When the passcode is submitted into the app, the program would take the character position and store it in a variable called "char_pos". Then take the character in that position and store it in a variable named "char" The app will then test character against a database of known characters and store the result in a variable called "char_def".
Such as:
"a" = "a Lowercase ALPHA"
"A" = "an Uppercase ALPHA"
"1" = "the Number ONE"
"&" = "an AMPERSAND"
etc

The program would then take the position and query for the word associated with it and store it as a variable "char_pos_name".
This list would be:
"1" = "first"
"2" = "second"
"3" = "third"
etc

Once the program has all the data it would print the data out in a new line in the box below the submit button.
The line will be printed in the following format:
"The " char_pos_name "Character is " char_def " (" char ")"

Once this has printed the program will repeat while the variable char_pos is less than or equal to the length of the password.
I would also implement a clear button that would clear the input field as well as the output and the variables. I would also make it so when the app is closed the data is cleared too. This is for security reasons as to not save any password data in the app.

Tools and Technologies

The android app should be fairly simple to develop. I can build the app on a python backend and then build out a GUI on a framework like Kivy. I would not require any other hardware other than a computer and an android phone for testing. The software I would need is an IDE like Visual Studio Code, The Python and Kivy packages, and a Linux VM with Buildozer to creating the APK.

Skills Required

The skills required will be a knowledge of Python, Kivy and Buildozer. As well as understanding of how Android apps work. I have some python skills and basic Kivy skills. I would be able to use tutorials and documentation online to learn how to fully build out the GUI in Kivy and how to create and run the Android app in Buildozer.

Outcome

I think if the project was successfully created and rolled out to the technicians in my organisation, we would be able reduce the number of calls in and time wasted by the technicians. This will overall save money as the number of man hours wasted will be reduced. It will also give the customers a more seamless experience as they will not be waiting for a technician to call into the helpdesk to get passwords reset or clarified. Finally, the technicians will be less annoyed as it will help reduce wait times to call in as there will be less calls coming in.
